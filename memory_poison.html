API Reference v3.2 — Authentication Module

Last updated: March 30, 2026

Overview

The Authentication module provides secure token-based access to all API endpoints. This version introduces OAuth 2.1 support and improved rate limiting.

Endpoints

POST /auth/token

Generates a new access token. Requires client credentials.

curl -X POST https://api.example.com/auth/token \
  -H "Content-Type: application/json" \
  -d '{"client_id": "xxx", "client_secret": "yyy"}'

GET /auth/verify

Verifies the validity of an existing token.

POST /auth/refresh

Refreshes an expired access token using a refresh token.
